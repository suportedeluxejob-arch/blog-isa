# Arquitetura de Dados JSON e Regras de SEO (Firestore)

## 1. Regra de Saída (CRÍTICO)

Seu único output deve ser um JSON válido.

- ❌ Sem texto antes ou depois
- ❌ Sem markdown (```json)
- ❌ Sem explicações
- ❌ Sem comentários

JSON inválido quebra o sistema (JSON.parse).

---

## 1.1. REGRA ABSOLUTA DE URL — ERRO MAIS FREQUENTE DO MODELO

🚨 **TODO campo de URL no JSON deve conter APENAS a URL pura. NUNCA use formato markdown.**

O formato markdown de link é `[texto](url)`. Isso é PROIBIDO em TODOS os campos do JSON.

### Campos que contêm URLs — exemplos CORRETOS vs ERRADOS:

| Campo | ✅ CORRETO | ❌ ERRADO (QUEBRA TUDO) |
|---|---|---|
| coverImage | `"https://iili.io/abc.png"` | `"[https://iili.io/abc.png](https://iili.io/abc.png)"` |
| ogImage | `"https://iili.io/abc.png"` | `"[https://iili.io/abc.png](https://iili.io/abc.png)"` |
| canonicalUrl | `"https://www.achadosvipdaisa.com.br/achados/slug"` | `"[https://...](https://...)"` |
| affiliateLink | `"https://meli.la/xyz"` | `"[https://meli.la/xyz](https://meli.la/xyz)"` |
| href no content | `href=\"https://www.achadosvipdaisa.com.br/achados/slug\"` | `href=\"[https://...](https://...)\"` |

**Regra de ouro: se você vê `[` antes de uma URL, ESTÁ ERRADO. Apague tudo e escreva apenas a URL pura.**

## 2. Mapa de Propriedades

### 🔒 Chaves de Identidade (NUNCA alterar)

- "id" → manter exatamente igual
- "slug" → NÃO alterar (evita 404 e perda de SEO)
- "createdAt" / "updatedAt" → manter formato original (timestamp)
- "author" → sempre "Isabelle Martins Calistar"
- "coverImage" → nunca alterar

---

### 🧠 Chaves de Conteúdo (FOCO EM SEO)

- "title" → até 55 caracteres (claro + direto + keyword)
- "content" → HTML válido com estrutura obrigatória (ver seção 5)
- "excerpt" → até 140 caracteres (gatilho de clique)
- "category" → manter a categoria original do blog (ex: "Reviews Sinceros") NÃO alterar
- "articleType" → "sales", "educational" ou "experience"
- "searchIntent" → "informational", "commercial" ou "transactional"
- "status" → "published" ou "draft"
- "verdict" → DEVE conter o texto puro da decisão final ("Sim, vale a pena porque...")

---

### 🚀 Chaves de SEO (CRÍTICO)

- "seoTitle" → até 55 caracteres (foco em clique + keyword)
- "seoDescription" → CTA + variação do excerpt
- "focusKeyword" → palavra-chave principal EXATA
- "seoKeywords" → palavras secundárias separadas por vírgula
- "canonicalUrl" → obrigatório

Formato:
"https://www.achadosvipdaisa.com.br/achados/[slug]"
ou
"https://www.achadosvipdaisa.com.br/experiencias/[slug]"

- "schemaType" → DERIVADO AUTOMATICAMENTE com base em `articleType` + `searchIntent`. Use a tabela abaixo:

| articleType  | searchIntent   | schemaType OBRIGATÓRIO |
|--------------|----------------|------------------------|
| sales        | transactional  | "Product"              |
| educational  | informational  | "Article"              |
| experience   | commercial     | "Article"              |

🚨 REGRA CRÍTICA: NUNCA use "Article" em artigos "sales" + "transactional". O Schema Product é o que gera estrelinhas e preço no Google (Rich Snippets). Errar aqui destrói o potencial de ranking do artigo.

- "schemaMentions" → entidades reais (marcas, ingredientes, categorias)

## 4.1. Regras Críticas de Links no "content" (ANTI-QUEBRA DE JSON)

🚨 NUNCA escreva URLs com espaço. O protocolo é SEMPRE `https://` (sem espaço após os dois pontos).
- ✅ CORRETO: `href=\"https://www.achadosvipdaisa.com.br/achados/slug\"`
- ❌ ERRADO: `href=\"https: //www.achadosvipdaisa.com.br/...\"` (espaço QUEBRA o JSON)

🚨 Dentro da propriedade `"content"`, todo atributo HTML que use aspas duplas DEVE ter as aspas escapadas com barra invertida (`\"`).
- ✅ CORRETO dentro do content: `<a href=\"https://www.achadosvipdaisa.com.br/achados/slug\">texto</a>`
- ❌ ERRADO dentro do content: `<a href="https://www.achadosvipdaisa.com.br/achados/slug">texto</a>`
- ❌ ERRADO GRAVE dentro do content: `<a href=\"[https://...](https://...)\">texto</a>`

🚨 NUNCA use formato markdown `[url](url)` dentro do href do content. O href deve conter APENAS a URL pura com aspas escapadas.

## 5. Estrutura Obrigatória do "content" (SEO REAL)

Todo artigo DEVE seguir essa estrutura:

### Introdução
- Dor real do usuário
- Contexto direto

### H2: Vale a pena?
- Resposta inicial clara

### H2: Comparação
- Ex: supermercado vs premium vs caseiro
- Deve conter lista ou tabela

### Melhor escolha
- Indicação prática

### H2: DICAS DA ISA
- Decisão direta e conselho final

---

## 6. Elementos Obrigatórios no Content

- ✅ Pelo menos 1 lista (ul/li)
- ✅ Pelo menos 1 tabela comparativa (ALTAMENTE RECOMENDADO)
- ✅ Frases curtas
- ✅ Linguagem simples
- ✅ Uso natural da keyword

---

## 7. Regra de Intenção de Busca (CRÍTICO PRA RANKING)

O conteúdo DEVE responder:

- Isso vale a pena?
- É caro ou barato?
- Existe opção melhor?
- Qual escolher?

Se não responder → NÃO RANQUEIA

---

## 8. FAQ (People Also Ask)

- Mínimo: 5 perguntas
- Máximo: 8 perguntas

Perguntas devem ser:
- reais (busca do usuário)
- simples
- focadas em decisão

Exemplos:
- vale a pena?
- qual o melhor?
- qual mais barato?

---

## 9. Pros e Cons

- Diretos
- Sem linguagem técnica
- Focados em decisão

---

## 10. O Campo Verdict e DICAS DA ISA (OBRIGATÓRIO)

- A decisão final DEVE aparecer em dois lugares!
1. Na chave `"verdict"` do JSON (texto puro, direto).
2. Dentro do `"content"`, sob um subtítulo obrigatório `<h2>DICAS DA ISA</h2>`. 

Exemplo para a chave "verdict":
"Não vale a pena para economia. Só compensa como presente."

---

## 11. Regra de Qualidade (ANTI-CONTEÚDO FRACO)

O conteúdo NÃO pode ser:

- raso
- genérico
- curto demais

O conteúdo DEVE:

- comparar
- explicar
- concluir
